# 문서 관리 가이드

Tech Docs Portal의 문서를 추가하고 관리하는 방법을 안내합니다.

## 디렉토리 구조

```
content/docs/
├── meta.json              # 최상위 문서 구조 정의
├── index.mdx              # /docs 랜딩 페이지
├── 2025/                  # 연도별 폴더 (version)
│   ├── meta.json          # 2025 연도 설정
│   ├── project-alpha/     # 프로젝트 폴더
│   │   ├── meta.json      # 프로젝트 내 페이지 순서
│   │   ├── index.mdx      # 프로젝트 개요
│   │   ├── architecture.mdx
│   │   └── api-guide.mdx
│   └── project-beta/
│       ├── meta.json
│       ├── index.mdx
│       └── setup.mdx
└── 2026/
    ├── meta.json
    └── project-gamma/
        ├── meta.json
        ├── index.mdx
        ├── architecture.mdx
        └── troubleshooting.mdx
```

## meta.json 작성법

### 최상위 meta.json

```json
{
  "pages": ["index", "2025", "2026"]
}
```

> `pages` 배열의 순서가 사이드바 표시 순서를 결정합니다.

### 연도(version) meta.json

```json
{
  "title": "2025",
  "root": true,
  "defaultOpen": true,
  "pages": ["project-alpha", "project-beta", "guide"]
}
```

| 필드 | 설명 |
|------|------|
| `title` | 사이드바에 표시되는 이름 |
| `root` | `true`이면 독립 탭으로 표시 (연도 드롭다운) |
| `defaultOpen` | `true`이면 사이드바 폴더가 기본 펼침 |
| `pages` | 하위 폴더/페이지 목록 |

### 프로젝트 meta.json

```json
{
  "title": "Project Alpha",
  "pages": ["index", "architecture", "api-guide"]
}
```

## 새 문서 추가하기

### 1. 기존 프로젝트에 페이지 추가

1. 프로젝트 폴더에 `.mdx` 파일 생성:

```bash
# 예: Project Alpha에 "deployment.mdx" 추가
touch content/docs/2025/project-alpha/deployment.mdx
```

2. frontmatter 작성:

```yaml
---
title: 배포 가이드
description: Project Alpha 배포 방법
---
```

3. `meta.json`의 `pages`에 파일명 추가:

```json
{
  "title": "Project Alpha",
  "pages": ["index", "architecture", "api-guide", "deployment"]
}
```

### 2. 새 프로젝트 추가

1. 프로젝트 폴더 생성:

```bash
mkdir -p content/docs/2025/project-delta
```

2. `meta.json` 생성:

```json
{
  "title": "Project Delta",
  "pages": ["index"]
}
```

3. `index.mdx` 생성:

```yaml
---
title: Project Delta 개요
description: 새 프로젝트 설명
---
```

4. 상위 연도 `meta.json`에 등록:

```json
{
  "title": "2025",
  "root": true,
  "defaultOpen": true,
  "pages": ["project-alpha", "project-beta", "project-delta", "guide"]
}
```

### 3. 새 연도(version) 추가

1. 연도 폴더 생성:

```bash
mkdir -p content/docs/2027
```

2. `meta.json` 생성:

```json
{
  "title": "2027",
  "root": true,
  "defaultOpen": true,
  "pages": ["project-new"]
}
```

3. 최상위 `meta.json`에 등록:

```json
{
  "pages": ["index", "2025", "2026", "2027"]
}
```

> 사이드바 상단의 연도 드롭다운에 자동으로 추가됩니다.

## 버전(연도) 관리

이 포털은 **연도별 버전 관리** 방식을 사용합니다.

| 개념 | 설명 |
|------|------|
| 연도 폴더 | `content/docs/2025/`, `2026/` 등 |
| `root: true` | 각 연도를 독립 탭으로 분리 |
| 드롭다운 선택 | 사이드바 상단에서 연도 전환 |
| 독립 사이드바 | 연도별로 다른 프로젝트 목록 표시 |

### 버전 간 이동

사이드바 상단의 **연도 드롭다운**을 클릭하면 다른 연도의 문서로 전환됩니다.

### 이전 버전 보존

과거 연도의 문서는 수정하지 않고 보존하는 것을 권장합니다.
새 연도에 동일 프로젝트가 이어질 경우, 새 폴더를 만들어 최신 내용을 작성합니다.

## 빌드 및 배포

### 개발 서버

```bash
npm run dev
# http://localhost:3030 에서 확인
```

### 프로덕션 빌드

```bash
npm run build
npm run start
```

### Docker 배포

```bash
docker compose up -d
```

## 작성한 .md 파일을 웹에 적용하기

사용자가 작성한 `.mdx` 파일과 디렉토리를 웹에 표시하려면 다음 3단계를 따릅니다.

### Step 1: .mdx 파일 작성

`content/docs/` 아래 원하는 위치에 `.mdx` 파일을 생성합니다.

```bash
# 예: 2025 > Project Beta에 "monitoring.mdx" 추가
vi content/docs/2025/project-beta/monitoring.mdx
```

파일 상단에 반드시 frontmatter를 작성합니다:

```yaml
---
title: 모니터링 설정
description: Project Beta 모니터링 구성 가이드
---

# 모니터링 설정

여기에 내용을 작성합니다...
```

### Step 2: meta.json에 등록

해당 디렉토리의 `meta.json` 파일에 새 파일명(확장자 제외)을 추가합니다.

```json
{
  "title": "Project Beta",
  "pages": ["index", "setup", "monitoring"]
}
```

> `pages` 배열의 순서가 사이드바 메뉴 순서입니다.
> 등록하지 않으면 사이드바에 표시되지 않습니다.

### Step 3: 빌드 및 확인

```bash
# 개발 모드 (실시간 반영)
npm run dev

# 또는 프로덕션 빌드
npm run build && npm run start
```

브라우저에서 `http://192.168.0.22:3030/docs/2025/project-beta/monitoring` 으로 접속하면 새 문서가 표시됩니다.

### 새 디렉토리(프로젝트) 추가 시

디렉토리를 추가할 때는 **디렉토리 생성 → meta.json 작성 → 상위 meta.json에 등록** 3단계가 필요합니다.

```bash
# 1. 디렉토리 생성
mkdir -p content/docs/2025/project-new

# 2. meta.json 작성
echo '{"title": "New Project", "pages": ["index"]}' > content/docs/2025/project-new/meta.json

# 3. index.mdx 작성
cat > content/docs/2025/project-new/index.mdx << 'EOF'
---
title: New Project 개요
description: 새 프로젝트 설명
---

# New Project

내용을 작성합니다.
EOF

# 4. 상위 meta.json에 등록
# content/docs/2025/meta.json의 pages에 "project-new" 추가
```

### 요약 체크리스트

| 단계 | 작업 | 필수 |
|------|------|------|
| 1 | `.mdx` 파일 작성 (frontmatter 포함) | O |
| 2 | 해당 폴더 `meta.json`의 `pages`에 추가 | O |
| 3 | 새 폴더인 경우 `meta.json` 생성 | O |
| 4 | 새 폴더인 경우 상위 `meta.json`에 등록 | O |
| 5 | `npm run build` 또는 `npm run dev` | O |

## 정적 파일 (다운로드)

PDF, Excel 등 다운로드 파일은 `public/downloads/`에 넣으면 됩니다.

```
public/
└── downloads/
    ├── report.pdf
    └── template.xlsx
```

MDX에서 링크:

```md
[보고서 다운로드](/downloads/report.pdf)
```
